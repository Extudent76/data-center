# Мониторинг дата-центра "Станкин"

Веб-приложение для отслеживания состояния оборудования машинных залов, управления информацией об арендаторах и мониторинга доступности ресурсов.

## Технологии

- **React 18+** - библиотека для построения пользовательского интерфейса
- **TypeScript** - типизированный JavaScript
- **MobX 6+** - реактивное управление состоянием
- **Recharts** - библиотека для визуализации данных
- **Gravity UI** - библиотека компонентов от Yandex
- **Axios** - HTTP клиент для работы с API
- **Vite** - инструмент сборки

## Установка

```bash
npm install
```

## Запуск

### Режим разработки с мок-данными (по умолчанию)

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

### Режим разработки с реальным API

Создайте файл `.env` в корне проекта:

```env
VITE_API_BASE_URL=http://your-api-url/api
VITE_USE_MOCK_DATA=false
```

Затем запустите:

```bash
npm run dev
```

## Сборка

```bash
npm run build
```

## Мок-данные

По умолчанию приложение использует мок-данные для разработки и тестирования. Мок-данные находятся в файле `src/api/mockData.ts` и соответствуют контракту API:

```typescript
{
  machineRoomsWidget: [...],  // Данные о доступности памяти машинных залов
  companiesWidget: {...},      // Данные о распределении арендаторов
  racksWidget: [...]          // Данные о стойках
}
```

## Структура проекта

```
src/
├── api/              # API клиент и типы запросов/ответов
├── stores/           # MobX хранилища
├── components/       # React компоненты
│   ├── common/       # Переиспользуемые компоненты
│   ├── layout/       # Компоненты макета (Header, Sidebar)
│   └── dashboard/    # Компоненты панели управления
├── pages/            # Страницы приложения
├── types/            # TypeScript типы и интерфейсы
├── utils/            # Вспомогательные функции
└── App.tsx           # Корневой компонент
```
